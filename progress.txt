## Codebase Patterns
- pnpmワークスペース: `packages/core/`, `packages/react/`, `examples/`
- TypeScript: strict mode, ESM, `verbatimModuleSyntax`, project references
- テスト: vitest, coverage v8, テストファイルは `packages/*/tests/`
- lint: ESLint flat config + typescript-eslint strict + @luma-dev/eslint-plugin-luma-ts (全ルール有効)
- format: Prettier
- scripts: `lint`, `lint-fix`, `test`, `test:run`, `coverage`, `typecheck`
- tsconfigのプロジェクト参照: テスト用は `tsconfig.test.json`, ルート設定用は `tsconfig.node.json`
- EffectResult: `Data.struct` で構造的等値性、型ガードは TypeScript 推論 (`no-explicit-return-is`)
- テンプレートリテラル内の式: `satisfies string` が必要 (`luma-ts/require-satisfies-in-tls`)
- `as unknown as T` 禁止 (`luma-ts/no-as-unknown-as`): delegate パターンで型安全に変換
- EffectStore: `StoreRuntime` インターフェースで ManagedRuntime に依存しない抽象化
- ManagedRuntime テスト: `ManagedRuntime.make(Layer.empty)` で空のランタイム作成
- EffectObserver: subscribe→run の順序で通知を漏らさない（store.subscribeを先に呼ぶ）
- React テスト: `@vitest-environment jsdom` コメント + `@testing-library/react` の `renderHook`/`render`
- React Context 型消去: `createContext<any>` + フック呼び出し時の `<R, E>` 型指定
- StoreRuntime: Fiber エラー型を `unknown` にすることで ManagedRuntime<R, E> と互換
- EffectProvider: EffectStore も Context で提供。runtime と同じライフサイクルで作成・dispose
- useEffectQuery: `useSyncExternalStore` + `createEffectObserver` の組み合わせ。key でキャッシュ共有
- Suspense フック: Promise を observer に自己購読させてサスペンド中でも resolve する。`EffectError<E>` で ErrorBoundary にキャッチ
- `Error` サブクラスの `cause` プロパティは ES2022 `Error.cause` と衝突するため `override` 必須
- React `Component` サブクラスの `render()` も `override` 必須（TypeScript strict + noImplicitOverride）
- useEffectMutation: runtime 直接利用 + `createSubscribable` でコンポーネントローカル管理。EffectStore 不使用
- mutation の observer fiber は直接キャンセル不要。mutation fiber interrupt → Fiber.await が Interrupt exit → ガードで無視
- useEffectStream: `Stream.runForEach` + `Effect.catchAll` でストリーム消費。interrupt は自然伝搬、typed errors のみ catch
- Stream テスト: `createControllableStream` ヘルパーで `emit/end/fail` を外部制御（`Stream.async` の Emit API を型安全にラップ）
- useEffectScope: `Effect.runSync` で Scope 操作（ManagedRuntime 不要）。`useEffect` 内でScope作成・close
- EffectBoundary: Suspense + ErrorBoundary 統合。`renderError` 未指定時はエラー伝搬。非 EffectError は常に親に伝搬
- EffectValue: React.memo + useRef で render prop の関数参照を安定化。effect も ref 化して memo 境界を維持
- ネストProvider: `EffectLayerContext` で親Layerを共有 → `Layer.merge(parentLayer, childLayer)` で自動合成。子サービスが優先
- QueryFilter: discriminated union (type: "exact" | "prefix" | "predicate" | "all") でフィルタパターン表現
- `luma-ts/no-date`: `Date.now()` 禁止。`getNow()` ラッパー + eslint-disable で対処（Temporal未対応環境）
- EffectStore config: `gcTime` (旧 gcGracePeriodMs) + `staleTime` (default: 0 = 常にstale)
- `resetEntry` vs `cleanupEntry`: resetEntry は subscriber 維持（clearCache用）、cleanupEntry は完全削除（GC用）
- EffectStore retry: `Schedule.tapOutput` でリトライ回数カウント。`RunOptions<E>` で schedule を渡し、`RetryState` を別 `Subscribable` で公開
- EffectStore optimistic: `setOptimistic(key, value)` → `OptimisticRollback.rollback()` で復元。version カウンタで競合検出（後勝ち）
- useEffectMutation: `MutationOptions.onMutate` が `OptimisticRollback` を返すと失敗時に自動ロールバック
- EffectStore tags: `RunOptions.tags` でエントリにタグ付け。`invalidateByTags(tags)` / `QueryFilter { type: "tags" }` でタグベース無効化
- useEffectMutation: `MutationOptions.invalidateTags` で成功時に関連クエリを自動再フェッチ
- SSR: `getServerSnapshot` は全フックでモジュールレベル定数（`() => initial`）。`noopSubscribe` も同様
- SSR: EffectProvider は `typeof window === "undefined"` で SSR 判定。SSR 時は null context で children をレンダー
- SSR テスト: `@vitest-environment node` + `renderToString` で実 SSR パスを検証。jsdom テストとは別ファイル
- CI: GitHub Actions `.github/workflows/ci.yml` で typecheck → lint → test (coverage) を実行。push/PR on main でトリガー
- SubscriptionRef カウンター: `useEffectRuntime<R, E>()` + `runtime.runFork` + `createSubscribable` + `useSyncExternalStore` パターン（フックの R 型制約を回避）
- Schema.TaggedClass: `Schema.TaggedClass<T>()("Tag", { fields })` で型定義。class instance spread 禁止（手動プロパティ指定）
- Layer合成: `ServiceLive.pipe(Layer.provide(DependencyLive))` で依存注入
- @effect/platform HttpClient: `HttpClientResponse.schemaBodyJson` は Scope を要求 → `Effect.scoped` が必要
- HttpClient 自己完結 Effect: `Effect.provide(FetchHttpClient.layer)` で R=never にして useEffectQuery に渡す
- Deferred ゲートパターン: Stream.paginateChunkEffect 内で `Deferred.await` → React ref 経由で `Deferred.succeed` するとユーザー操作でストリーム進行を制御可能
- `Deferred.make<undefined>()` を使う（`void` は `@typescript-eslint/no-invalid-void-type` でエラー）
- v8 coverage: switch の同一処理 case は統合する（CI/localでカバレッジ差異が出るフレーク防止）
- EffectBoundary renderError: `Cause<unknown>` を受け取る（型消去）。`instanceof` で AppError に runtime narrowing

- Layer.scoped: `Layer.scoped(Tag, Effect.gen(function* () { ... yield* Effect.acquireRelease(...); ... }))` でリソースのライフサイクルをLayerスコープ（Provider）に紐づける
- withConnection パターン: `Effect.ensuring` で接続取得→使用→解放を安全に管理。エラー時も確実にリリース

- **今後のイテレーションへの学び:**
  - 楽観的更新の競合検出にはバージョンカウンタが最もシンプル。参照等価性（`Object.is`）だと同値の異なるオブジェクトで誤判定する
  - observer fiber が settle した時も version をインクリメントしないと、楽観的更新→effect 完了の順序で rollback が古い値に戻してしまう
  - `let` 変数 + closure + `vi.waitFor` の組み合わせでは TypeScript の制御フロー解析が `never` に推論することがある。配列に push + `.at(0)` パターンで回避
  - `onMutate` の戻り値型は `OptimisticRollback | undefined`（`void` は ESLint `no-invalid-void-type` でエラー）
  - `@typescript-eslint/no-unnecessary-type-parameters`: 型パラメータが1箇所しか使われない場合は `unknown` に置き換える（`setOptimistic<A>(key, value: A)` → `setOptimistic(key, value: unknown)`）
