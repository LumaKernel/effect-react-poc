## Codebase Patterns
- pnpmワークスペース: `packages/core/`, `packages/react/`, `examples/`
- TypeScript: strict mode, ESM, `verbatimModuleSyntax`, project references
- テスト: vitest, coverage v8, テストファイルは `packages/*/tests/`
- lint: ESLint flat config + typescript-eslint strict + @luma-dev/eslint-plugin-luma-ts (全ルール有効)
- format: Prettier
- scripts: `lint`, `lint-fix`, `test`, `test:run`, `coverage`, `typecheck`
- tsconfigのプロジェクト参照: テスト用は `tsconfig.test.json`, ルート設定用は `tsconfig.node.json`
- EffectResult: `Data.struct` で構造的等値性、型ガードは TypeScript 推論 (`no-explicit-return-is`)
- テンプレートリテラル内の式: `satisfies string` が必要 (`luma-ts/require-satisfies-in-tls`)
- `as unknown as T` 禁止 (`luma-ts/no-as-unknown-as`): delegate パターンで型安全に変換
- EffectStore: `StoreRuntime` インターフェースで ManagedRuntime に依存しない抽象化
- ManagedRuntime テスト: `ManagedRuntime.make(Layer.empty)` で空のランタイム作成

## 2026-03-03 - EffectStore コア実装 (タスク 1.4)
- 実装した内容:
  - `EffectStore` インターフェース: `getSubscribable`, `run`, `invalidate`, `subscribe`, `getSnapshot`, `dispose`
  - `StoreRuntime` インターフェース: ManagedRuntime互換の最小ランタイムインターフェース
  - `createEffectStore` ファクトリ関数: ランタイム + 設定を受け取り EffectStore を生成
  - キーベースのキャッシュ管理: 各キーごとに `MutableSubscribable<EffectResult<A, E>>` で状態管理
  - 状態遷移: Initial → Pending → Success/Failure、Success → Refreshing → Success
  - Fiber管理: 同一キー再実行時に前のFiberをinterrupt、dispose時に全Fiberをinterrupt
  - GCタイマー: 設定可能な猶予期間（デフォルト30秒）、unsubscribe後にエントリを削除
  - invalidate: 最後に実行したEffectを再実行
  - 23テスト (状態遷移、Fiber管理、invalidate、subscribe、GCタイマー、複数キー、dispose)
- 変更したファイル:
  - packages/core/src/EffectStore.ts (新規)
  - packages/core/src/index.ts (re-export追加)
  - packages/core/tests/EffectStore.test.ts (新規)
  - tasks/01-foundation.md (タスク1.4を完了マーク)
- カバレッジ: 100% → 100% (EffectStore.ts 100%)
- **今後のイテレーションへの学び:**
  - `luma-ts/no-as-unknown-as`: `as unknown as T` は禁止。delegate パターン（新しいオブジェクトでラップ）で型安全に変換
  - `Fiber.interruptFork` は fire-and-forget の interrupt に適する。`Fiber.interrupt` は await する場合に使用
  - `Exit.match` で Success/Failure を分岐するのが最もクリーン
  - ManagedRuntime の `runFork` は Effect を fork して Fiber を返す。テストでは `ManagedRuntime.make(Layer.empty)` で空のランタイムを作成
  - GCタイマーのテストでは `vi.useFakeTimers()` / `vi.advanceTimersByTime()` が効果的
  - 非同期Effect完了の待機には `vi.waitFor()` が便利
